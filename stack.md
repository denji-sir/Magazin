# Стек технологий (Правила и Обоснование)

Этот документ определяет основной набор инструментов для разработки e-commerce проекта. Каждая библиотека выбрана для обеспечения баланса между скоростью разработки и стабильностью.

---

## 🎨 Frontend (Клиентская часть)

1. **React**
   - **Для чего:** Основной JS-фреймворк.
   - **Почему:** Стандарт индустрии. Компонентный подход позволяет легко собирать интерфейс из переиспользуемых частей (карточки, кнопки, меню).

2. **Vite**
   - **Для чего:** Сборщик проекта и сервер разработки.
   - **Почему:** Обеспечивает мгновенный запуск и быструю перезагрузку приложения при изменениях кода.

3. **Mantine (UI, Hooks, Form)**
   - **Для чего:** Единая экосистема визуальных компонентов, хуков и управления формами.
   - **Почему:** Позволяет не тратить время на верстку с нуля. Включает готовые модальные окна, поля ввода, кнопки и встроенную валидацию данных.

4. **React Router DOM**
   - **Для чего:** Навигация между страницами.
   - **Почему:** Позволяет перемещаться по разделам сайта (Каталог, Корзина, Админка) без перезагрузки всей страницы в браузере.

5. **Axios**
   - **Для чего:** HTTP-клиент для связи с сервером.
   - **Почему:** Удобный и надежный способ отправлять данные на бэкенд и получать ответы.

6. **Framer Motion**
   - **Для чего:** Библиотека для сложных анимаций.
   - **Почему:** Добавляет интерфейсу плавность и «премиальность»: появление элементов, переходы между страницами и hover-эффекты.

7. **TanStack Table**
   - **Для чего:** Мощный движок для таблиц.
   - **Почему:** Используется в админ-панели для удобной работы с большими списками товаров и заказов (сортировка, фильтрация).

8. **Lucide React**
   - **Для чего:** Набор современных иконок.
   - **Почему:** Огромная библиотека легких и красивых иконок, которые легко настраивать.

9. **React Hot Toast**
   - **Для чего:** Всплывающие уведомления.
   - **Почему:** Простой способ мгновенно сообщить пользователю о действии (например, «Товар добавлен в корзину»).

---

## ⚙️ Backend (Серверная часть)

1. **Python & Flask**
   - **Для чего:** Серверный каркас и создание API.
   - **Почему:** Легкий и понятный фреймворк, который идеально подходит для быстрой разработки серверной логики.

2. **Flask-SQLAlchemy**
   - **Для чего:** Работа с базой данных (ORM).
   - **Почему:** Позволяет управлять данными в БД через Python-код, избавляя от необходимости писать громоздкие SQL-запросы.

3. **Flask-JWT-Extended**
   - **Для чего:** Безопасная авторизация.
   - **Почему:** Реализует механизм токенов (JWT) для входа в систему и защиты приватных данных пользователя или админа.

4. **Flask-CORS**
   - **Для чего:** Разрешение кросс-доменных запросов.
   - **Почему:** Необходим, чтобы браузер не блокировал общение между фронтендом и бэкендом, запущенными на разных портах.

5. **Werkzeug Security**
   - **Для чего:** Защита паролей.
   - **Почему:** Используется для надежного хеширования паролей перед их сохранением в базу данных.

6. **SQLite**
   - **Для чего:** Хранилище данных (База данных).
   - **Почему:** База данных «в одном файле». Не требует установки и настройки сервера БД, что делает проект максимально переносимым.
