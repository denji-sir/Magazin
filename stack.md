# Стек технологий (Правила и Обоснование)

Проект строится по клиент-серверной архитектуре. Кастомная админка реализована на React — Django Admin в качестве основного интерфейса не используется.

```
React + Mantine
пользовательский интерфейс + кастомная админка
        ↓ Axios / REST API (JSON)
Django + Django REST Framework
серверная логика + авторизация + база данных
        ↓
SQLite
```

---

## 🎨 Frontend

| Библиотека | Для чего |
|---|---|
| **React** | Основной JS-фреймворк. Всё — от карточки товара до кастомной админки — компоненты на React. |
| **Vite** | Сборщик. Быстрый старт и мгновенная перезагрузка при изменении кода. |
| **Mantine UI** | Визуальные компоненты: кнопки, карточки, модалки, формы, бейджи, меню — не верстаем с нуля. |
| **Mantine Hooks** | Вспомогательные хуки: адаптивность, управление состоянием UI, размер экрана. |
| **Mantine Form** | Управление формами и встроенная валидация (логин, регистрация, оформление заказа, добавление товара). |
| **React Router DOM** | Навигация между страницами без перезагрузки (каталог, корзина, личный кабинет, админка). |
| **Axios** | HTTP-клиент. Отправляет запросы к Django API и получает JSON-ответы. |
| **Framer Motion** | Анимации: появление карточек, переходы между страницами, hover-эффекты, открытие модальных окон. |
| **TanStack Table** | Мощный движок таблиц для кастомной админки (товары, заказы, пользователи — с сортировкой и фильтрацией). |
| **Lucide React** | Иконки: корзина, поиск, профиль, редактирование, статусы — единый визуальный язык. |
| **React Hot Toast** | Всплывающие уведомления: «товар добавлен», «заказ оформлен», «ошибка входа». |

## ⚙️ Backend

| Библиотека | Для чего |
|---|---|
| **Python** | Основной язык серверной части. |
| **Django** | Серверный фреймворк. Управляет роутингом, моделями и бизнес-логикой. |
| **Django REST Framework** | Превращает Django в полноценный REST API: сериализаторы, viewsets, обработка запросов и ответов в JSON. |
| **Django ORM** | Работа с базой данных через Python-код вместо SQL (модели: User, Product, Order, Cart и др.). |
| **Django Auth** | Встроенная система пользователей: роли, пароли, пермиссии. Основа для разделения admin / user. |
| **Simple JWT** | Токенная авторизация (JWT). Выдает access/refresh токены при входе, защищает приватные API-маршруты. |
| **django-cors-headers** | Разрешает запросы от React-приложения к Django API, когда они работают на разных портах. |
| **SQLite** | База данных в одном файле. Не требует установки сервера — идеально для диплома. |

## 📋 API-контракт (что Django должен предоставить React-у)

### Пользователи
- `POST /auth/register/` — регистрация
- `POST /auth/login/` — авторизация, получение JWT-токенов
- `GET /auth/me/` — текущий пользователь
- `PATCH /auth/me/` — редактирование профиля
- `POST /auth/token/refresh/` — обновление токена

### Товары
- `GET /products/` — список (поиск, фильтрация, сортировка)
- `GET /products/{id}/` — карточка товара
- `POST /products/` — создание *(только admin)*
- `PATCH /products/{id}/` — редактирование *(только admin)*
- `DELETE /products/{id}/` — удаление *(только admin)*

### Категории
- `GET /categories/` — список категорий
- `POST /categories/` — создание *(только admin)*
- `PATCH /categories/{id}/` — редактирование *(только admin)*
- `DELETE /categories/{id}/` — удаление *(только admin)*

### Корзина
- `GET /cart/` — корзина текущего пользователя
- `POST /cart/` — добавить товар
- `PATCH /cart/{id}/` — изменить количество
- `DELETE /cart/{id}/` — удалить из корзины
- `DELETE /cart/clear/` — очистить корзину

### Заказы
- `POST /orders/` — оформить заказ
- `GET /orders/` — заказы пользователя
- `GET /orders/{id}/` — детали заказа
- `GET /admin/orders/` — все заказы *(только admin)*
- `PATCH /orders/{id}/status/` — изменить статус *(только admin)*
- `POST /orders/{id}/cancel/` — отмена заказа

### Псевдооплата
- `POST /payment/create/` — создать платежную сессию
- `POST /payment/process/` — имитация обработки
- `GET /payment/{id}/status/` — статус оплаты
- `POST /payment/{id}/retry/` — повторить попытку

### ПВЗ (Пункты выдачи заказов)
- `GET /pvz/` — список ПВЗ
- `POST /pvz/` — создать *(только admin)*
- `PATCH /pvz/{id}/` — редактировать *(только admin)*
- `DELETE /pvz/{id}/` — удалить *(только admin)*
- `PATCH /pvz/{id}/toggle/` — включить/отключить *(только admin)*

## 🔒 Правила защиты

> Административные API доступны **только** пользователям с ролью `admin`.

**Обычный пользователь НЕ может:**
- создавать, редактировать и удалять товары;
- просматривать все заказы;
- менять статусы заказов;
- управлять пользователями, ПВЗ и категориями.

---

## 📝 Django Admin — только для разработки

Django Admin **не является** основным интерфейсом управления. Используется только технически:
- проверка данных в БД;
- создание первого администратора;
- отладка моделей.

Основной административный интерфейс — кастомная панель на **React + TanStack Table + Mantine**.
